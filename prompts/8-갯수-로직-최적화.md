운동을 더 여러가지로 대응하고싶다.  
이 앱의 컨셉은 '홈트전용 ai 헬스트레이너'다. 집에서 맨몸으로 해볼 수 있는 push-up, sit-up, squat를 추가할 것이다.

해당 운동들의 개수 counting 알고리즘을 고도화하기위해, @videos/ 경로에 영상들을 추가해두었다.

@videos/index.json 은 각 영상들에 대해서, 개수를 count하면 되는 시점(second)들은 countSeconds 필드에 저장되어있고, threshold만큼의 second 오차가 발생해도 괜찮다. 예를 들어, countSeconds가 12이고 threshold가 0.2라면, 11.8이상 12.2이하 시점에 개수를 count했다면 성공적으로 알고리즘을 구현한 것으로 판단한다.

내 목표는 우리 앱의 관절 포인트 기반 counting 로직을 최적화하기위해, 각 영상들의 관절포인트 움직임 데이터를 미리 추출해두고, 이를 분석하는  
 python script를 만든 뒤 detect 로직 관련한 변수를 조금씩 바꿔가며 모든 countSeconds에 잘 count하고, countSeconds가 아닐땐 count하지 않는  
 최적의 변수값들을 찾는 것이다.

python으로 실험하는 이유는 swift와 다르게, cli 환경에서 빠르게 반복실행하며 일단 최적화 값을 찾아두고, 이 값들을 그대로 swift 코드에  
 입력하고싶기 때문이다. 이를 위해 적절한 코드 구조가 필요할 것이며, swift 환경에서 사용될 관절 포인트 지점들과 python 환경에서의 지점들이  
 일치해야만할 것이다.

이 실험을 진행하기위한 구체적인 계획을 세우기위해 조금이라도 모호하거나 구체화해야할 것이 있다면 내게 알려달라.
