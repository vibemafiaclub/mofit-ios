# Mission — Mofit

## 비전

**폰 하나로 운동이 교정되는 경험을 만든다.**

별도 웨어러블·PT·스마트 미러 없이 iPhone 카메라만으로 실시간 자세 분석과 코칭을 제공하는 AI 트레이너. 운동 습관을 만들고 싶지만 장비·비용·전문가 접근성에 막힌 개인이 타깃.

## 가치 제안

- **운동 트래킹이 1차 가치.** 스쿼트 자동 카운팅 + 자세 교정이 핵심. AI 코칭은 상단 부가 가치.
- **즉시성.** 앱 켜고 → 폰 세워두고 → 손바닥 1초 → 운동 시작. 로그인·설정 없이 첫 스쿼트까지 20초.
- **개인화.** 사용자 프로필(성별/키/몸무게/체형/목표) + 최근 7일 운동 이력을 Claude 컨텍스트로 전달해, 일반 피트니스 앱의 템플릿 조언이 아닌 **내 데이터 기반** 피드백을 준다.
- **오프라인 우선.** 비로그인 상태에서는 완전 로컬(SwiftData). 로그인해도 운동 자체는 로컬에서 완결되고 서버는 저장/AI 프록시 역할만.

## 비즈니스 모델 (가정)

- **단계 0 (현재, YC-targeted MVP)**: 무료. 사용자·사용량·리텐션 검증이 목표. BM 확정 전.
- **단계 1 (가설)**: AI 코칭 호출 수·고급 지표(주간 리포트·운동별 폼 점수) 기반 프리미엄. 기본 카운팅은 영구 무료.
- **단계 2 (가설)**: B2B — 헬스장·PT 숍이 고객 운동 데이터를 확인할 수 있는 대시보드.

현재 단계에서 BM 결정을 강제하지 않는다. 아이디에이션/설득력 검토 시에는 "무료 MVP가 리텐션을 얻고 있는가"를 최상위 지표로 본다.

## 타깃 사용자

- **핵심 페르소나**: 운동 습관을 만들고 싶지만 헬스장·PT가 부담스러운 20~30대. 집에서 맨몸 운동 위주로 시작.
- **2차 페르소나 (가설)**: 재활/자세 교정이 필요한데 매번 병원·전문가를 찾기 힘든 사용자.

페르소나 세부사항은 `persuasion-data/personas/` 하위 파일이 캐노니컬 소스. 이 문서는 지향점만 기록한다.

## 원칙 (철학)

- **속도 최우선.** 외부 의존성 최소. 설정·스키마·화면은 필요한 최소치만.
- **AI 에이전트 + CLI로 완결.** 기획·운영·분석·배포 대부분이 에이전트 한 명으로 끝나야 한다. 인간 개입이 꼭 필요한 지점은 `docs/user-intervention.md` 에 명시.
- **증거 기반 의사결정.** "있으면 좋을 것 같아서" 기능 추가 금지. 설득력 검토(persuasion-review) 결과와 페르소나 피드백이 붙지 않은 요구사항은 `tech-critic-lead` 가 거부한다.
- **단일 사용자 MVP.** 소셜·랭킹·커뮤니티는 의도적 제외. 앞으로 당분간도.

## 의도적 제외 범위 (MVP)

`docs/adr.md` ADR-009 참조. 간단히:

- 카메라 미인식 안내, 세트 완료 진동, 폰 위치 가이드
- 앱 강제종료 시 운동 데이터 복구
- SwiftData 마이그레이션
- 스쿼트 외 운동 (UI에도 노출하지 않음, ADR-017)
- 로컬 → 서버 데이터 마이그레이션
