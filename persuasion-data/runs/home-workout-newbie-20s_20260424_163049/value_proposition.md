# Mofit — 폰 하나로 자세가 교정되는 AI 스쿼트 트레이너

## 1. 가치제안 요약 (랜딩페이지 카피)

### 메인 카피

**"폰 세워두고 손바닥 1초. 스쿼트가 자동으로 세어집니다."**

iPhone 카메라만으로 스쿼트를 자동으로 세고, 관절 각도로 실시간 자세를 보여줍니다. 별도 장비·회원가입 없이, 앱 켠 지 20초 안에 첫 카운트가 올라갑니다.

### 서브 카피

- **자동 카운팅 (스쿼트 전용, MVP 단계)**: Apple Vision으로 hip–knee–ankle 각도 분석. 서있음(>160°) → 앉음(<100°) → 서있음 = 1 rep 인정.
- **로그인 없이 바로 시작**: 설치 → 30초 온보딩 → 즉시 카운트. 기록은 내 폰 로컬(SwiftData)에만 저장, 외부 전송 0.
- **AI 코칭 샘플은 로그인 전에도 미리 본다**: 코칭 탭에 들어가면 로그인 전에도 샘플 피드백 2종이 먼저 보여서, 로그인할 가치가 있는지 확인 후 결정할 수 있음.
- **광고 없음 · 결제 없음**: MVP 단계는 전면 무료. 기본 카운팅·기록·자세 오버레이는 **영구 무료 유지 계획** (경계는 §3 참조).

### 타깃 시나리오

> "헬스장 3개월 등록했는데 2주 못 갔어요. PT는 회당 7~10만원, 유튜브 홈트는 내 폼이 맞는지 모르겠어요."

Mofit은 바로 이 지점을 겨냥합니다. 폰을 벽에 기대 세우고 손바닥을 카메라에 1초 보여주면 5초 카운트다운 후 추적이 시작됩니다. 스쿼트가 끝날 때까지 폰을 만지지 않아도 됩니다.

### 현재 지원 범위 (솔직한 고지)

- **스쿼트만 실제 판정 동작함.** 푸쉬업/싯업 버튼은 **"준비중" 배지 + 회색 톤(opacity 0.4)** 으로 노출되며, 탭하면 "현재는 스쿼트만 지원합니다" 토스트만 나오고 트래킹으로 진입하지 않습니다.
- 스쿼트 외 운동의 실제 판정 로직은 검증 후 순차 추가 예정. **지금 설치하면 스쿼트 전용 앱으로 쓰게 됩니다.**
- 푸쉬업/싯업 로드맵 시점은 아직 공개하지 않습니다 (확실하지 않은 일정은 약속하지 않음).

### 솔직한 성능 고지

- 내부 개발 중 관측한 감각으로는 권장 환경(밝은 실내, 전신이 프레임에 들어오는 거리, 일상복)에서 대체로 안정적으로 카운트됩니다. **다만 공개 가능한 정량 정확도 수치(예: %)는 현재 확보되어 있지 않습니다.** 베타 이후 측정해 구체 수치로 갱신할 계획.
- 조도 낮음 / 카메라 너무 가까움 / 하체 일부가 프레임 밖 등의 조건에서는 카운트가 튀거나 멈추는 경우가 있습니다 — 현 버전은 "왜 안 되는지"를 안내하는 UI가 아직 얕습니다 (§5 참고).

## 2. 구체적 기능 설명 (미팅 / 데모 전달 시)

### 2.1 온보딩 (최초 1회, 약 30초)

성별 → 키 → 몸무게 → 체형(마른/보통/근육질/통통한) → 목표(체중 감량/근력 증가/체형 개선). 회원가입·이메일 요구 없음. 입력은 전부 폰 로컬(SwiftData)에 저장.

### 2.2 홈 탭

- 오늘의 기록 요약 (세트 수, 총 rep, 시간)
- "운동 시작" 형광초록 버튼
- 운동 종류 선택 (ExercisePicker):
  - **스쿼트**: 활성 (정상 동작)
  - **푸쉬업 / 싯업**: opacity 0.4 + "준비중" 배지. 탭 시 트래킹 미진입 + 토스트

### 2.3 트래킹 화면 (카메라)

- **스쿼트 판정**: `VNDetectHumanBodyPoseRequest`로 hip-knee-ankle 각도 분석. 서있음 → 앉음 → 서있음 = 1 rep.
- **손바닥 판정**: `VNDetectHumanHandPoseRequest`로 5개 손가락이 1초 펴진 상태 감지 → 세트 시작/종료 신호.
- 관절 포인트가 화면에 오버레이로 표시.
- 프리뷰 30fps, 분석 15fps 샘플링 (배터리 절약 + 스쿼트 속도에 충분).

### 2.4 기록 탭

- 상단: 최근 7일 가로 스크롤 날짜바
- 선택 날짜의 세션 리스트 (운동 종류, 세트 수, 총 rep, 시간, 시작 시각)
- 스와이프 삭제

### 2.5 AI 코칭 탭

**비로그인 분기 (설치 직후 바로 확인 가능)**: 로그인하지 않아도 코칭 탭에 진입하면 "로그인하면 이런 피드백을 받습니다" 섹션과 함께 **샘플 피드백 카드 2종**이 먼저 노출됩니다. 각 샘플은 **숫자 + 요일/시점 + 구체 행동 제안** 3요소를 포함하도록 고정:

- **샘플 A (운동 전 동기부여)**: *"지난 주 3일 운동 · 총 78회 스쿼트 · 일평균 26회. 수요일엔 38회 최다였고 금요일 0회였네요. 오늘은 수요일 페이스 회복해서 30회 이상 도전해보세요."*
- **샘플 B (운동 후 피드백)**: *"오늘 3세트 총 32회. 1세트 14회 → 2세트 10회 → 3세트 8회. 세트별 감소폭 4회로 피로도 자연스러운 곡선. 내일은 2세트째에서 쉬는 시간 20초 늘려보세요."*
- 각 카드 하단에 "※ 예시 피드백 (실제 데이터 기반으로 매번 다름)" 고지.

**로그인 분기 (활성)**: 하루 2회 호출(운동 전 동기부여 / 운동 후 피드백, 자정 리셋). **앱 → 자체 서버 `/coaching/request` → Anthropic Claude API** 경유 (API 키는 앱 바이너리에 없음). 전달 컨텍스트: 사용자 프로필 + 최근 7일 요약(운동일수, 총 세션, 총 rep, 일평균 rep) + 일별 추이(rep/세트/세트당 평균). API 실패 시 일일 횟수 차감 안 함.

### 2.6 인증 (선택)

- 이메일 + 비밀번호 가입/로그인 (Supabase + JWT, Keychain 저장).
- 로그인 유저: 프로필/운동 기록/AI 피드백이 서버(Supabase)에도 저장.
- 비로그인 유저: 모든 데이터 로컬(SwiftData)에만 저장, 외부 전송 0.
- **주의**: 비로그인 상태로 쌓은 로컬 기록은 로그인 후 서버로 **이관되지 않음** (의도된 설계, ADR-013). 동일 기기에서 재로그아웃 시 로컬 데이터는 남아있음.

## 3. 가격 / 무료 범위 경계 (솔직한 고지)

### 단계 0 (현재, MVP)

- 전 기능 무료, 광고 없음.

### 단계 1 가설 (수개월 후, 미확정)

- **영구 무료로 유지하려는 기능** (변경 시 공지): 스쿼트 자동 카운팅, 세션 기록 저장/조회, 관절 각도 오버레이, "준비중"으로 표시된 운동의 향후 자동 카운팅.
- **과금 후보 영역**: 하루 2회를 초과하는 AI 코칭 호출, 주간 리포트·장기 추이 분석, 커스텀 자세 가이드 — **아직 확정된 플랜 아님**. 변경 시 인앱 공지 + 이 페이지에 이력 유지.

## 4. 프라이버시 / 데이터

- 카메라 영상은 **어디에도 저장·전송되지 않음**. 온디바이스 Vision 분석 후 즉시 폐기.
- 비로그인: 모든 데이터 로컬. 로그인: Supabase 서버 저장 + Claude API 서버 프록시.
- API 키는 앱 바이너리에 없음 (서버 프록시).

## 5. 리스크 · 한계 · 아직 얕은 부분 (선제 고지)

현재 MVP 단계에서 **의도적으로 얕게 남긴 영역**입니다. 이 지점들은 "지금은 몰래 잘 되는 척" 하지 않고 미리 고지합니다.

1. **rep 카운트 정량 정확도 공개 미제공** — 권장 환경/비권장 환경에서의 수치(정확도 %, 미검출률) 확보 전. 베타 이후 측정해 갱신.
2. **손바닥 인식 실패 폴백 부재** — 손바닥이 1초 인식되지 않으면 카운트다운이 시작되지 않습니다. 현재는 화면 탭 등의 대체 트리거가 없음. 이 경우 사용자는 앱을 재진입하거나 조명/각도를 조정해야 함.
3. **실패 가이드 UI 얕음** — "카메라에 전신이 안 들어왔다" / "조도가 낮다" 같은 실패 원인 안내 UI는 현재 없음.
4. **스쿼트 외 운동 미제공** — §1 "현재 지원 범위" 참조. 로드맵 시점은 미공개.
5. **앱스토어 출시 전 단계** — 별점·리얼 리뷰 등의 외부 신뢰 신호가 아직 없습니다. 현재 시점에서 공개 가능한 신뢰 신호는: (a) 이 문서 자체의 범위/한계 투명 고지, (b) 설치·사용 **비용이 0**이라 "망설이지 않고 설치해 직접 확인" 가능, (c) 카메라 영상 비저장/비전송 + API 키 서버 분리 등 프라이버시 설계 근거. 베타 사용자 피드백이 쌓이면 후기 발췌를 랜딩에 반영할 계획.
6. **크래시/데이터 복구** — 앱 크래시 시 진행 중 세트는 복구되지 않음 (MVP 스코프 외).

## 6. 설치 후 첫 10분 시나리오 (기대치 교정)

1. 설치 → 카메라 권한 허용 → 온보딩 30초
2. 홈 탭에서 "스쿼트" 탭 → 카메라 뷰 진입
3. 폰을 벽에 기대거나 거치 → 전신이 프레임에 들어오도록 2~3m 거리
4. 손바닥을 카메라 방향으로 1초 들기 → 5초 카운트다운 → 추적 시작
5. 스쿼트 수행 → rep이 올라가는 것을 확인
6. 끝낼 때 다시 손바닥 1초 → 세트 종료, 결과 화면
7. (선택) 코칭 탭에서 샘플 피드백 2종 확인 후 로그인 여부 결정
