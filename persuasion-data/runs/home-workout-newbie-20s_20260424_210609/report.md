# 최종 판정

**실패 — keyman_drop (5a 단계에서 종결)**

keyman(개인 소비자, 홈트 초급자)은 초기 응답에서 `decision: drop`, `confidence: 55`로 판정. 성사 조건(`decision == convince_stakeholders` AND `confidence > 75`)을 충족하지 않아 5b 이하 단계는 수행되지 않았다. 페르소나 정의상 `stakeholders: []`이므로 조직 설득 경로 자체가 없고, keyman 한 명의 drop이 곧 전체 실패로 직결된다.

keyman이 제시한 결정적 이탈 사유는 (1) 스쿼트 전용이라는 포지셔닝 한계, (2) 카운트 정확도 정량 지표 부재, (3) 앱스토어 미출시에 따른 외부 신뢰 신호 0, 이 세 가지가 핵심. 페르소나의 `reject_triggers` 중 "rep 카운트가 자주 틀림 → 신뢰도 붕괴" 항목과 `communication_style`("리뷰·앱스토어 별점으로 의사결정")이 그대로 발화에 반영됨.

# 실행 리스크

**낮음 (가정적)** — 시뮬 상에서는 실패했지만, 만약 통과했다면 리스크 강도는 낮음이다. `decision_authority: full`이고 조직 실무자/협업자가 없는 개인 소비자 페르소나이므로, keyman의 accept가 곧 실제 설치·사용으로 이어지는 구조. 조직 내 "keyman accept이지만 도입 무산" 같은 실행 리스크는 애초에 존재하지 않는다.

다만 **사용 지속 리스크**는 별도. 페르소나가 "3일 써보고 아니면 삭제"라는 이탈 패턴을 명시했기 때문에, 설치까지 설득했더라도 첫 3일 안에 카운트 오류·스쿼트 외 운동 부재 경험이 1회라도 나오면 즉시 리텐션 붕괴. 이는 "설득 성공 ≠ 정착"인 개인 소비자 특유의 2차 관문이다.

# 가치제안 개선 포인트

1. **스쿼트 전용 한계와 포지셔닝 정렬 실패** — 세션 1건(5a). 대표 발화: *"나는 스쿼트 카운터만 찾아 검색하지 않는다. 푸쉬업·런지·플랭크 등 맨몸 운동 루틴 전반을 돌릴 수 있는 앱을 원한다. (...) 3일 써보고 다른 운동 안 되면 그대로 삭제 경로."* → "홈트 앱"을 찾는 소비자의 기대치와 "스쿼트 전용" 현실 사이 간극. 단일 운동으로는 B2C 개인 사용자 설득에 구조적 벽.

2. **카운트 정확도 정량 지표 부재** — 세션 1건. 대표 발화: *"'내부 감각으로는 안정적'은 설득 재료가 아니다. 내 reject_trigger 1번이 'rep 카운트가 자주 틀림 → 신뢰도 붕괴'다. 정량 데이터가 없으니 직접 설치해서 확인하는 리스크를 내가 떠안아야 한다."* → 정확도 %·조건별 실측 공개 필요. 리스크 이전을 소비자에게 떠맡기면 보수적 페르소나는 drop.

3. **외부 신뢰 신호 부재 (앱스토어 미출시 + 리뷰 0 + 추천 0)** — 세션 1건. 대표 발화: *"나는 리뷰·별점으로 앱을 고르는 사람이다. TestFlight 베타도 준비 중 단계라면, 내가 '얼리 테스터'가 될 이유는 없다."* → 페르소나의 `communication_style`과 정면 충돌. 베타 테스터 유인책, 인플루언서 리일, 사전 사용자 후기 축적이 구조적으로 필요.

4. **온보딩 마찰 (6단계 · 3~4분)** — 세션 1건. 대표 발화: *"나는 첫 30초 안에 뭔가 되길 기대한다. (...) 체형·목표·코치 스타일까지 6단계는 설치 직후 피로감."* → 프로파일 구매 맥락과 일치. 온보딩 단계 지연/스킵 가능성 고려.

5. **비로그인→로그인 기록 미이관(ADR-013) 설계 모순** — 세션 1건. 대표 발화: *"무료로 3일 체험해서 쌓은 기록이 로그인하는 순간 서버로 안 옮겨진다면, '로그인=손해'로 느껴진다. 이 설계 자체가 내 행동 경로(일단 무료 체험 → 만족하면 계정 연결)와 충돌."* → 체험→전환 퍼널의 결정적 마이그레이션 포인트가 끊김. 제품 설계 레벨 재검토 필요.

6. **AI 코칭 차별점 검증 불가 + 과금 불확실성** — 세션 1건. 대표 발화: *"실제 코칭 퀄리티가 샘플 수준인지 generic한 ChatGPT 조언 수준인지 검증할 방법이 없다. (...) '영구 무료'라던 카운팅 영역까지 유료화로 밀릴 가능성은 없는가?"* → 차별점(AI 코칭)이 로그인 뒤로 숨겨져 있고, 무료 영구 보장도 "미확정"으로 표시돼 신뢰 훼손.

7. **엣지 케이스·복구 시나리오 공백** — 세션 1건. 대표 발화: *"권장 환경(밝은 실내·2~3m·전신) 벗어나면 카운트가 얼마나 튀는가? (...) 크래시 시 진행 중 세트 복구 없음 — 홈트 중 전화·앱 전환만으로도 세트 날리는 거 아닌가?"* → 조명·거리 민감도 자료, 백그라운드 복귀 시 세트 유지 설계가 보수적 소비자 설득 재료.
