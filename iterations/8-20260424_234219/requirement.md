# Mofit — iPhone 카메라만으로 스쿼트를 자동으로 세는 앱

## 1. 가치제안 요약 (랜딩페이지 카피)

### 메인 카피

**"폰 세워두고 손바닥 1초 또는 화면 탭. 스쿼트가 자동으로 세어집니다."**

iPhone 카메라만으로 스쿼트를 자동으로 세고, 관절 각도로 실시간 자세를 보여줍니다. 별도 장비·회원가입 없이, 앱 켠 지 1분 안에 첫 카운트가 올라갑니다.

### 서브 카피

- **스쿼트 전용 자동 카운터**: Apple Vision으로 hip–knee–ankle 각도 분석. 서있음(>160°) → 앉음(<100°) → 서있음 = 1 rep 인정. 분석 15fps 샘플링(배터리 절약 + 스쿼트 속도에 충분).
- **로그인 없이 바로 시작**: 설치 → 온보딩 → 즉시 카운트. 기록은 내 폰 로컬(SwiftData)에만 저장, 외부 전송 0.
- **시작·종료 트리거 이중화**: 손바닥 1초 OR **화면 탭**. 손바닥 인식이 안 되는 각도/조도에서도 탭 한 번으로 카운트다운 시작.
- **안 세어지는 이유를 인앱에서 바로 안내 (ADR-018)**: 트래킹 중 양쪽 다리가 프레임에 안 잡히거나, 조도가 낮아 하체 조인트 평균 confidence가 임계치 미만으로 3초 지속되면 상단에 반투명 배너로 "전신이 프레임에 들어오는지 확인하세요" / "조명이 어두울 수 있어요" 힌트 표시. 5초 grace 후, rep이 한 번이라도 카운트되면 이후 숨김 (주의 분산 방지).
- **rep 단위 autosave (ADR-009 2026-04-24)**: 세트 시작 시점에 `WorkoutSession` 을 즉시 로컬 DB 에 insert, 매 rep 이 카운트될 때마다 `repCounts` 스냅샷을 `modelContext.save()`. 운동 중 전화·앱 전환·배터리 방전·크래시가 나도 **여기까지 센 rep 은 기록 탭에 그대로 남음**. 비로그인 유저 한정 (Phase 1).
- **비로그인 AI 코칭 샘플이 내 온보딩 값·로컬 기록 기반으로 즉시 개인화 (iter 7, 2026-04-24)**: 코칭 탭 비로그인 분기에 진입하면 성별/키/목표/체형 + 최근 7일 로컬 `WorkoutSession` 을 `CoachingSampleGenerator` 가 결정론적 템플릿에 대입해 운동 전/후 샘플 2장을 즉시 생성. "여성 / 160cm / 감량 목표" · "첫 세션 10회 3세트" 같은 문구가 **내 입력값**으로 채워져 노출. 서버/Claude API 호출 없음.
- **광고 없음 · 결제 없음**: MVP 단계는 전면 무료. 기본 카운팅·기록·자세 오버레이는 영구 무료 유지 계획.

### 타깃 시나리오

> "헬스장 3개월 등록했는데 2주 못 갔어요. PT는 회당 7~10만원, 유튜브 홈트는 내 폼이 맞는지 모르겠어요."

Mofit은 바로 이 지점을 겨냥합니다. 폰을 벽에 기대 세우고 손바닥을 카메라에 1초 보여주거나 화면을 탭하면 5초 카운트다운 후 추적이 시작됩니다. 스쿼트가 끝날 때까지 폰을 만지지 않아도 되고, 끝낼 때도 탭 또는 손바닥으로 세트 종료가 가능합니다.

### 현재 지원 범위 (솔직한 고지)

- **Mofit은 스쿼트 전용 카운터입니다.** 푸쉬업·런지·싯업·플랭크 등 다른 운동은 제공하지 않습니다.
- 홈 탭 시작 버튼은 "스쿼트 시작" 하나뿐이며, 운동 종류 선택 UI 자체가 없습니다 (ADR-017).
- 다른 운동 확장에 대한 일정은 약속하지 않습니다. 지금 설치하면 **스쿼트를 자동으로 세는 앱**으로 쓰게 됩니다.

### 솔직한 성능 고지

- 내부 개발 중 관측한 감각으로는 권장 환경(밝은 실내, 전신이 프레임에 들어오는 거리 2~3m, 일상복)에서 대체로 안정적으로 카운트됩니다. **다만 공개 가능한 정량 정확도 수치(예: 오카운트율 %, 조건별 미검출률)는 현재 확보되어 있지 않습니다.** 베타 이후 실제 사용 데이터로 측정해 구체 수치로 갱신할 계획.
- 조도 낮음 / 카메라 너무 가까움 / 하체 일부가 프레임 밖 등의 조건에서는 카운트가 튀거나 멈추는 경우가 있습니다. 이 경우 **§2.3의 진단 힌트 배너가 원인(프레임 아웃 / 저조도)을 인앱에서 안내**합니다.

## 2. 구체적 기능 설명 (미팅 / 데모 전달 시)

### 2.1 온보딩 (최초 1회, 약 3~4분)

성별 → 키 → 몸무게 → 체형(마른/보통/근육질/통통한) → 목표(체중 감량/근력 증가/체형 개선) → 코치 스타일. 총 6단계, 입력값 허용/경고 범위 검증 포함. 회원가입·이메일 요구 없음. 입력은 전부 폰 로컬(SwiftData)에 저장.

### 2.2 홈 탭

- 좌상단: "Mofit" 타이틀. 우상단: 프로필 편집 버튼.
- **"스쿼트 시작" 단일 버튼(형광초록)**. 탭 시 즉시 트래킹 화면 진입. 운동 종류 선택 UI 없음 (ADR-017).
- 오늘의 기록 요약: "세트 N · 스쿼트 N회 · N분" (기록 없으면 "첫 운동을 시작해보세요!")
- 운동 종료 후 복귀 시 폭죽 효과.

### 2.3 트래킹 화면 (카메라)

- **스쿼트 판정**: `VNDetectHumanBodyPoseRequest`로 hip-knee-ankle 각도 분석. 왼/오 다리 중 하나 기준. 서있음(>160°) → 앉음(<100°) → 서있음 = 1 rep.
- **손바닥 판정**: `VNDetectHumanHandPoseRequest`로 5개 손가락이 1초 펴진 상태 감지 → 세트 시작/종료 신호.
- **화면 탭 대체 트리거**: idle/tracking 상태에서 ZStack 전체 `.onTapGesture` 활성. 손바닥이 안 잡히는 각도·조도에서도 탭 한 번으로 동일 경로 실행. idle 서브카피: "손바닥을 보여주거나 화면을 탭하세요". tracking 서브카피: "끝낼 땐 화면을 탭하거나 손바닥을 보여주세요".
- **진단 힌트 오버레이 (ADR-018)**: 트래킹 시작 5초 유예 후, 양쪽(left/right) hip/knee/ankle 3조인트 모두 3초간 미검출이면 `.outOfFrame` 배너("전신이 프레임에 들어오는지 확인하세요 · 2~3m 거리 권장") 노출. 그 외 하체 조인트 평균 confidence < 0.5 가 3초 지속이면 `.lowLight` 배너("조명이 어두울 수 있어요 · 실내 조명을 밝혀주세요"). 우선순위 outOfFrame. 한 세션 내 rep 한 번 카운트되면 힌트 숨김 + 이후 재표시 금지. 배너는 반투명 상단 고정, 하단 종료 버튼 시야 가리지 않음.
- **rep 단위 autosave (ADR-009 2026-04-24)**: `countdown → tracking` 전이 시점에 빈 `WorkoutSession` 을 즉시 SwiftData insert + save. 이후 rep 이 증가할 때마다 해당 세션의 `repCounts` 배열에 스냅샷 append + `endedAt = .now` 갱신 + `try? modelContext.save()`. 세트 정상 종료(`completeSet`) 시 한 번 더 최종 save. 결과: 스쿼트 중 **전화 수신·앱 전환·OS 강제 종료·크래시**가 나도 "다음 번 앱 진입 시 기록 탭에 그때까지 센 rep 수가 남아있음". 비로그인 유저 한정 (Phase 1).
- 관절 포인트가 화면에 오버레이로 표시.
- 프리뷰 30fps, 분석 15fps 샘플링.

### 2.4 기록 탭

- 상단: 최근 7일 가로 스크롤 날짜바.
- 선택 날짜의 세션 리스트 (세트 수, 총 rep, 시간, 시작 시각).
- 스와이프 삭제.
- 기록 없는 날: "이 날은 운동 기록이 없어요".
- **0rep 세션은 숨김**: autosave 도입 부수효과로, rep 이 단 한 번도 카운트되지 않은 세션이 로컬 DB 에 남을 수 있음. 기록 탭 쿼리 단에서 `totalReps > 0` 메모리 필터로 자동 숨김 (사용자 스와이프 삭제 불필요).

### 2.5 AI 코칭 탭

**비로그인 분기 (iter 7 동적 생성, 2026-04-24)**: 로그인하지 않아도 코칭 탭에 진입하면 "이런 피드백을 받게 됩니다" 섹션과 함께 **온보딩 값 + 최근 7일 로컬 세션 기반으로 결정론적으로 생성된 운동 전/후 샘플 2장**이 노출됩니다. 처리 전부 폰 안에서 (Foundation-only, 네트워크·랜덤 없음). 예시:

- **샘플 Pre (기록 있음 · 감량 목표)**: *"여성/160cm/감량 목표. 최근 7일 3회 운동으로 총 42회 스쿼트를 쌓으셨습니다. 오늘도 12회를 겨냥해 칼로리 소모 페이스를 이어가세요."*
- **샘플 Pre (기록 없음 · 근력 목표)**: *"남성/175cm/근력 목표. 첫 세트는 10회로 시작하고 하단에서 1초 멈추면 자극이 커집니다."*
- **샘플 Post (기록 있음 · 체형 목표)**: *"체형 개선 목표 기준 최근 세션 3세트 총 28회(1세트 12회 → 2세트 10회 → 3세트 6회). 최근 7일 누적 28회. 내일은 각 세트 마지막 2회에서 하강 속도를 3초로 늘려 하체 라인을 다잡아보세요."*
- 카드 하단 고지 그대로: "※ 예시 피드백 (실제 데이터 기반으로 매번 다름)".
- **프로필이 아직 없으면(온보딩 미완)** 샘플 대신 "온보딩을 먼저 완료해주세요" CTA 카드로 대체 (하드코딩 샘플 폴백 없음).
- 베이스 템플릿: `goal(3) × 기록유무(2) = 6개`. 요일/bodyType/성별/키는 문자열 인터폴레이션. 템플릿 개수 상한 10개(초과 시 재승인).

**로그인 분기 (활성)**: 하루 2회 호출(운동 전 동기부여 / 운동 후 피드백, 자정 리셋). **앱 → 자체 서버 `/coaching/request` → Anthropic Claude API** 경유 (API 키는 앱 바이너리에 없음). 전달 컨텍스트: 사용자 프로필 + 최근 7일 요약(운동일수, 총 세션, 총 rep, 일평균 rep) + 일별 추이(rep/세트/세트당 평균). API 실패 시 일일 횟수 차감 안 함. 서버는 DB 유일성 제약으로 1일 2회를 강제.

### 2.6 인증 (선택)

- 이메일 + 비밀번호 가입/로그인 (Supabase + JWT, Keychain 저장).
- 로그인 유저: 프로필/운동 기록/AI 피드백이 서버(Supabase)에도 저장.
- 비로그인 유저: 모든 데이터 로컬(SwiftData)에만 저장, 외부 전송 0.
- **주의**: 비로그인 상태로 쌓은 로컬 기록은 로그인 후 서버로 **이관되지 않음** (의도된 설계, ADR-013).

## 3. 가격 / 무료 범위 경계 (솔직한 고지)

### 단계 0 (현재, MVP)

- 전 기능 무료, 광고 없음.

### 단계 1 가설 (수개월 후, 미확정)

- **영구 무료로 유지하려는 기능**: 스쿼트 자동 카운팅, 세션 기록 저장/조회, 관절 각도 오버레이, 진단 힌트, rep 단위 autosave, 비로그인 코칭 샘플 동적 생성.
- **과금 후보 영역**: 하루 2회를 초과하는 AI 코칭 호출, 주간 리포트·장기 추이 분석, 커스텀 자세 가이드 — **아직 확정된 플랜 아님**.

## 4. 프라이버시 / 데이터

- 카메라 영상은 **어디에도 저장·전송되지 않음**. 온디바이스 Vision 분석 후 즉시 폐기.
- 비로그인: 모든 데이터 로컬. 로그인: Supabase 서버 저장 + Claude API 서버 프록시.
- API 키는 앱 바이너리에 없음 (서버 프록시).
- 비로그인 코칭 샘플 생성은 Foundation-only 온디바이스 처리. 네트워크 호출 없음.

## 5. 리스크 · 한계 · 아직 얕은 부분 (선제 고지)

현재 MVP 단계에서 **의도적으로 얕게 남긴 영역**입니다. 이 지점들은 "지금은 몰래 잘 되는 척" 하지 않고 미리 고지합니다.

1. **rep 카운트 정량 정확도 공개 미제공** — 권장 환경/비권장 환경에서의 수치(오카운트율 %, 미검출률) 확보 전. 베타 이후 측정해 갱신. 내부적으로도 실패 케이스 로깅 파이프라인이 아직 없음.
2. **진단 힌트는 2종으로 고정 (ADR-018)** — 현재 `.outOfFrame` / `.lowLight` 두 케이스만 커버. "카메라가 너무 가까움", "복장 때문에 하체 윤곽이 안 잡힘" 같은 세부 원인은 구분하지 않음. 2종의 임계치도 v1 경계값이며 실사용 데이터 누적 후 튜닝.
3. **스쿼트 외 운동은 영구적으로 제공되지 않을 수 있음** — MVP는 스쿼트 전용 포지셔닝이 확정됨(ADR-017). "홈트 앱"을 기대하고 설치하면 어긋남.
4. **앱스토어 출시 전 단계** — 별점·리얼 리뷰 등의 외부 신뢰 신호가 아직 없습니다. TestFlight 베타 배포도 준비 중. README에 스크린샷도 아직 없음. 현재 시점에서 공개 가능한 신뢰 신호는: (a) 이 문서 자체의 범위/한계 투명 고지, (b) 설치·사용 **비용이 0**이라 "망설이지 않고 설치해 직접 확인" 가능, (c) 카메라 영상 비저장/비전송 + API 키 서버 분리 등 프라이버시 설계 근거.
5. **비로그인 → 로그인 로컬 기록 미이관 (ADR-013)** — 비로그인 상태에서 쌓은 기록은 로그인해도 서버로 옮겨지지 않음. "3일 무료로 써서 쌓인 기록이 로그인 시 통째로 리셋되는 것처럼 보일 수 있다"는 유저 불신 이슈가 아직 남아 있음.
6. **rep autosave 는 비로그인 한정 (Phase 1 scope)** — 로그인 유저는 현행대로 "세트 정상 종료 시 서버 저장" 만 유지. 로그인 중 interruption 으로 rep 이 날아가는 케이스는 Phase 2 에서 서버/로컬 하이브리드 전략과 함께 재검토. 현재 로그인 유저에게는 "interruption 내성" 고지 안 함.
7. **rep autosave 는 세트 시작 이후만 방어** — 트래킹 화면 진입은 했지만 `countdown → tracking` 전이 전에 앱이 죽으면 여전히 기록 0. 전이 이전 구간(손바닥 1초 대기, 5초 카운트다운 중)은 autosave 대상 아님.
8. **비로그인 코칭 샘플은 결정론적 템플릿** — iter 7 에서 온보딩 값 + 로컬 세션을 문자열 인터폴레이션으로 꽂아 넣는 수준. "자세가 왜 무너졌는지, 다음 세트에 정확히 무엇을 바꿔야 하는지" 같은 **해석적 피드백**은 아직 로그인 분기의 Claude API 호출에서만 제공. 비로그인 샘플과 로그인 실제 응답 사이에는 해석 깊이의 gap 이 남음.
9. **AI 코칭 차별점 "실사용 검증" 은 여전히 로그인 뒤** — 비로그인 샘플은 이제 개인화되어 "ChatGPT 템플릿 느낌" 이탈은 완화. 그러나 실제 Claude API 응답 퀄리티(내 데이터를 정말 읽고 구체 피드백을 주는지)는 여전히 이메일·비밀번호 가입 + 7일치 기록 축적 이후에만 확인 가능.
10. **온보딩 이탈 시 코칭 탭이 CTA 한 장으로 비어있음** — 온보딩을 중단한 유저가 코칭 탭에 직접 먼저 들어오면 "온보딩을 먼저 완료해주세요" CTA 한 장만 보임. 차별점 증거 카드가 전혀 노출되지 않아, 온보딩 회피 유저가 가치를 체감할 경로가 없음.
11. **카메라 권한 거부 유저 동선** — 카메라 권한을 처음에 거부하면 홈 "스쿼트 시작" 이후 플로우가 회복되지 않음. 권한 재요청 UI·설정 딥링크가 별도로 없음. 3일 컷 페르소나의 설치 직후 "카메라 권한 없으면 아무것도 못함" reject_trigger 가 실제 구현에 그대로 남아있을 가능성.

## 6. 설치 후 첫 10분 시나리오 (기대치 교정)

1. 설치 → 카메라 권한 허용 → 온보딩 3~4분
2. 홈 탭에서 "스쿼트 시작" 버튼 탭 → 카메라 뷰 진입
3. 폰을 벽에 기대거나 거치 → 전신이 프레임에 들어오도록 2~3m 거리
4. 손바닥을 카메라 방향으로 1초 들거나 **화면을 탭** → 5초 카운트다운 → 추적 시작
5. 스쿼트 수행 → rep이 올라가는 것을 확인. 프레임 밖이거나 조도가 낮으면 상단 배너로 원인 안내 (ADR-018). 중간에 전화·크래시가 나도 그때까지 센 rep 은 기록 탭에 자동 저장 (ADR-009 2026-04-24).
6. 끝낼 때 다시 손바닥 1초 또는 **화면 탭** → 세트 종료, 결과 화면
7. (선택) 코칭 탭에서 **내 온보딩 값·오늘 기록이 문구에 박힌 샘플 피드백 2장** 확인 → "generic 아니네" 감지 후 로그인 여부 결정
